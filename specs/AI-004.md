# Spec: AI-004 工作分类标签生成

## 功能需求

AI 自动为每条记录生成工作分类标签，支持自定义标签体系，并支持按标签筛选记录。

## 验收条件

### AC1: 每条记录自动分类
- Given 用户启动自动捕获
- When AI 分析截图完成
- Then 记录自动附加分类标签（如：开发、会议、写作、学习等）
- Given 用户查看记录列表
- When 记录有标签
- Then 显示标签徽章

### AC2: 支持自定义标签体系
- Given 用户打开设置界面
- When 查看"标签分类"部分
- Then 可编辑自定义标签列表（每行一个标签）
- Given 用户保存自定义标签
- When 下次 AI 分析时
- Then AI 从自定义标签体系中选择最匹配的标签

### AC3: 可按标签筛选记录
- Given 今日有多条记录且带有不同标签
- When 用户选择某个标签筛选
- Then 仅显示包含该标签的记录
- Given 用户清除筛选
- Then 显示所有记录

## 技术设计

### 数据库变更

```sql
-- records 表新增 tags 列
ALTER TABLE records ADD COLUMN tags TEXT; -- JSON 数组格式，如 ["开发", "Rust"]

-- settings 表新增 tag_categories 列
ALTER TABLE settings ADD COLUMN tag_categories TEXT; -- JSON 数组格式，如 ["开发", "会议", "写作", "学习"]
```

### 后端 API

| 命令 | 描述 |
|-----|------|
| `get_records_by_tag` | 按标签筛选记录 |
| `get_all_tags` | 获取所有使用过的标签 |
| `get_default_tag_categories` | 获取默认标签分类列表 |

### AI Prompt 扩展

修改 DEFAULT_ANALYSIS_PROMPT 增加 tags 字段：

```json
{
  "current_focus": "编写 Rust 后端代码",
  "active_software": "VS Code",
  "context_keywords": ["Rust", "Tauri", "异步编程"],
  "tags": ["开发", "编程"]
}
```

### 默认标签分类

```json
["开发", "会议", "写作", "学习", "研究", "沟通", "规划", "文档", "测试", "设计"]
```

## 风险

- AI 生成的标签可能不一致（使用自定义标签体系约束）
- 标签数量过多影响 UI（限制每条记录最多 3 个标签）
