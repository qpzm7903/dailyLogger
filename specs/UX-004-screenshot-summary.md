# UX-004 截图分析摘要显示

**版本**: v1.40.0
**优先级**: MEDIUM

## 功能需求

截图自动分析完成后，在截图缩略图下方或悬停时显示一段简短的应用摘要（来自 AI 分析结果的前 80 字），让用户快速了解每张截图的内容，无需点击展开详情。

**当前问题**:
- 截图分析完成后，用户必须点击每张截图才能查看 AI 分析内容
- 历史记录列表中截图条目与手动记录条目视觉区分度低
- 用户无法快速扫描截图列表了解一天的活动概况

## 不在范围内

- 不修改 AI 分析的内容格式或 Rust 后端
- 不修改截图存储逻辑
- 不实现全文搜索（属于独立功能）

## 接口定义

### 摘要显示规格

```typescript
// 摘要提取规则
function extractSummary(analysisContent: string, maxLength = 80): string {
  // 1. 解析 AI 分析 JSON 内容
  // 2. 提取 description 或 summary 字段
  // 3. 截断至 maxLength 字符，末尾加 "…"
}
```

### 展示方式

| 场景 | 显示方式 |
|------|---------|
| 截图列表默认视图 | 缩略图右侧/下方显示 1 行摘要文字 |
| 分析中 | 显示"分析中…"骨架占位符 |
| 分析失败 | 显示"分析失败"灰色文字 |
| 无分析内容 | 不显示摘要区域（不留空白） |

### 摘要文字样式

```
text-sm text-gray-400 line-clamp-1 truncate
```

## 验收条件（Given/When/Then）

### AC1 - 分析完成后显示摘要

- Given 截图已完成 AI 分析
- When 用户查看历史记录中的截图条目
- Then 在截图缩略图旁边显示不超过 80 字的摘要文字

### AC2 - 分析中显示占位符

- Given 截图已触发但 AI 分析尚未返回结果
- When 用户查看该截图条目
- Then 显示动态骨架占位符（而非空白区域）

### AC3 - 摘要不撑开列表高度

- Given 摘要文字较长
- When 显示在列表中
- Then 摘要限制为单行（line-clamp-1），超出部分用省略号截断

### AC4 - 无摘要时不影响布局

- Given 截图条目没有 AI 分析内容（如旧数据）
- When 显示在历史记录列表中
- Then 条目高度与有摘要的条目一致，无异常空白

## 技术约束

- 只使用 TailwindCSS，无内联样式
- 摘要提取函数须有单元测试（覆盖正常、空内容、JSON 解析失败三种情况）
- 不修改 SQLite schema（摘要从现有 content 字段实时提取）
- `npm run test` 全部通过
