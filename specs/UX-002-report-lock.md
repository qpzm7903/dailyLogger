# UX-002 报告生成按钮状态互锁

**版本**: v1.40.0
**优先级**: HIGH

## 功能需求

当用户点击任意报告生成按钮（日报、周报、月报）后，所有报告生成按钮立即进入禁用+加载状态，直到当前生成任务完成或失败。防止用户重复触发导致的并发 API 调用和界面混乱。

**当前问题**:
- 用户可在报告生成过程中多次点击，触发多个并发 AI 请求
- 生成过程中没有加载指示，用户不知道是否在处理中
- 多次点击可能导致多份报告写入文件系统

## 不在范围内

- 不修改报告生成的 Rust 后端逻辑
- 不修改报告内容格式
- 不新增取消功能（UX-011 再考虑）

## 接口定义

### 组件状态

```typescript
// 在报告相关组件中
const isGenerating = ref(false)  // 全局生成锁

// 按钮 disabled 条件
const isButtonDisabled = computed(() => isGenerating.value)
```

### 加载状态样式规范

| 状态 | 按钮样式 | 图标 |
|------|---------|------|
| 空闲 | 正常样式 | 无 |
| 生成中（当前按钮） | `opacity-75 cursor-not-allowed` | 旋转 spinner |
| 生成中（其他按钮） | `opacity-50 cursor-not-allowed` | 无 |
| 生成完成 | 短暂绿色闪烁（500ms）→ 恢复正常 | ✓ |

## 验收条件（Given/When/Then）

### AC1 - 点击后立即锁定

- Given 报告生成界面处于空闲状态
- When 用户点击任意报告生成按钮
- Then 所有报告生成按钮在 100ms 内变为禁用状态，被点击的按钮显示 spinner

### AC2 - 生成完成后解锁

- Given 报告正在生成中
- When 后端返回成功结果
- Then 所有按钮恢复可用，显示 success Toast

### AC3 - 生成失败后解锁

- Given 报告正在生成中
- When 后端返回错误
- Then 所有按钮恢复可用，显示 error Toast（符合 UX-001 规范）

### AC4 - 防止重复调用

- Given 报告正在生成中
- When 用户尝试再次点击按钮（通过键盘或自动化工具）
- Then `invoke()` 不被调用，操作被静默忽略

## 技术约束

- 只使用 TailwindCSS，无内联样式
- 锁定状态通过 Vue `ref` 管理，不引入新的状态管理库
- 前端测试须覆盖：锁定逻辑、解锁逻辑（成功/失败两种情况）
- `npm run test` 全部通过
