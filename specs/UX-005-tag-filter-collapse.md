# UX-005 标签过滤条折叠

**版本**: v1.40.0
**优先级**: LOW

## 功能需求

当历史记录过滤区域的标签数量超过一行可显示的宽度时，超出部分折叠为 "+N 个标签" 按钮。点击该按钮展开显示所有标签。避免大量标签撑开界面高度，影响历史记录列表的可见区域。

**当前问题**:
- 标签数量多时（如 10+ 个），过滤条占用大量垂直空间
- 历史记录列表可见条目数减少
- 标签排列杂乱，难以快速找到目标标签

## 不在范围内

- 不修改标签的创建和管理逻辑
- 不修改过滤的业务逻辑（选中的标签仍然生效）
- 不实现标签搜索功能

## 接口定义

### 折叠逻辑规格

```typescript
// 折叠阈值：超过 1 行时触发折叠
// 实现方式：测量容器宽度，计算可显示的标签数

interface TagFilterState {
  allTags: string[]
  selectedTags: string[]
  isExpanded: boolean
  visibleCount: number      // 当前宽度下可显示的标签数
  overflowCount: number     // 溢出标签数 = allTags.length - visibleCount
}
```

### 展示规则

| 条件 | 显示内容 |
|------|---------|
| 标签数 ≤ 一行容量 | 全部显示，无折叠按钮 |
| 标签数 > 一行容量 + 折叠收起状态 | 显示前 N 个标签 + "+M 个标签" 按钮 |
| 展开状态 | 全部标签 + "收起" 按钮 |

### "+N 个标签" 按钮样式

```
text-sm text-blue-400 hover:text-blue-300 cursor-pointer whitespace-nowrap
```

**注意**：即使被折叠的标签中有选中的标签，折叠按钮也应有视觉提示（如显示选中数量）。

## 验收条件（Given/When/Then）

### AC1 - 标签不超出时不折叠

- Given 过滤条中标签数量在一行内可完整显示
- When 用户查看历史记录过滤区域
- Then 所有标签正常显示，不出现折叠按钮

### AC2 - 标签超出时自动折叠

- Given 过滤条中标签数量超过一行容量
- When 用户查看历史记录过滤区域
- Then 超出部分隐藏，显示 "+N 个标签" 按钮（N 为隐藏的标签数）

### AC3 - 点击展开显示所有标签

- Given 过滤条处于折叠状态
- When 用户点击 "+N 个标签" 按钮
- Then 所有标签展开显示，按钮变为 "收起"

### AC4 - 折叠标签中有选中项时给出提示

- Given 某个被折叠隐藏的标签处于选中状态
- When 过滤条处于折叠状态
- Then "+N 个标签" 按钮有视觉标记（如蓝色高亮或显示 "已选N"）

### AC5 - 响应窗口宽度变化

- Given 用户调整应用窗口宽度
- When 宽度发生变化
- Then 折叠逻辑重新计算，自动更新可见标签数量

## 技术约束

- 只使用 TailwindCSS，无内联样式
- 不使用 `overflow: hidden` 直接截断（视觉不佳），使用 Vue 计算属性动态切片
- 宽度计算使用 ResizeObserver 或 `useElementSize` composable
- 前端测试须覆盖折叠逻辑计算函数
- `npm run test` 全部通过
