# 系统托盘菜单完善规格

## 功能需求
完善系统托盘功能，提供更便捷的快捷操作菜单，提升后台运行时的用户体验。

## 优化范围
1. 托盘右键菜单增强（启动/停止捕获、生成日报、打开设置）
2. 托盘图标状态指示（运行中/暂停）
3. 托盘工具提示显示当前状态
4. 双击托盘图标显示主窗口

## 不在范围内
- 不修改托盘基础功能（已有最小化到托盘）
- 不新增系统通知
- 不修改主窗口逻辑

## 验收条件（Given/When/Then）

### AC1 - 托盘菜单
- Given 应用运行中
- When 右键点击托盘图标
- Then 显示菜单包含：显示窗口、启动/停止捕获、生成日报、设置、退出选项

### AC2 - 状态切换
- Given 自动捕获正在运行
- When 点击菜单中的"停止捕获"
- Then 停止捕获，菜单项变为"启动捕获"

### AC3 - 图标状态
- Given 自动捕获正在运行
- When 查看托盘图标
- Then 图标显示活动状态（如绿色圆点或变化）

### AC4 - 工具提示
- Given 应用运行中
- When 鼠标悬停在托盘图标上
- Then 显示工具提示：当前状态（捕获中/已暂停）、今日记录数

### AC5 - 双击唤醒
- Given 主窗口已隐藏
- When 双击托盘图标
- Then 显示主窗口

## 技术约束
- 使用 Tauri v2 tray 插件 API
- 跨平台兼容（Windows/macOS/Linux）
- Rust 测试：`cargo test` 须通过

## 接口定义

### 后端函数
- `get_tray_menu_state()` - 获取托盘菜单当前状态
- `toggle_auto_capture()` - 切换自动捕获状态

### Tauri 事件
- `tray-menu-update` - 托盘菜单状态更新事件
