# 日报生成模板优化规格

## 功能需求
改进 AI 日报生成的输出格式和内容质量，提供更结构化、更易读的日报模板。

## 优化范围
1. 优化日报 Markdown 格式结构
2. 支持工作时间统计
3. 改进 AI 分析 Prompt，提取更关键信息
4. 支持待办事项提取
5. 优化代码块和链接格式

## 不在范围内
- 不修改日报生成核心逻辑
- 不新增数据库字段
- 不支持自定义模板（属于 AI-003）

## 验收条件（Given/When/Then）

### AC1 - 格式结构
- Given 当日有记录
- When 生成日报
- Then 输出包含：标题、今日概览、主要成果、工作时间线、待跟进事项五个标准章节

### AC2 - 时间统计
- Given 当日有多条记录
- When 生成日报
- Then 今日概览章节显示工作时长估算（基于首尾记录时间差）

### AC3 - 关键信息提取
- Given 记录包含技术内容
- When 生成日报
- Then 主要成果章节提取关键决策、技术要点、完成事项

### AC4 - 待办提取
- Given 记录包含待办相关内容（如 "TODO"、"待处理"、"需要" 等）
- When 生成日报
- Then 待跟进事项章节自动提取并列出

### AC5 - 格式优化
- Given 生成的日报内容
- When 用户查看
- Then 代码块正确格式化，链接可点击，列表层级清晰

## 技术约束
- 修改 Rust synthesis 模块的 Prompt 构建
- 确保向后兼容现有日报格式
- Rust 测试：`cargo test` 须通过

## 接口定义

### 后端函数
- `generate_daily_summary()` - 优化 Prompt 构建

### Prompt 模板要点
```
生成日报时请遵循以下格式：
1. 今日概览：工作时长、记录条数、截图数量
2. 主要成果：按重要性排序的关键成果（3-5 条）
3. 工作时间线：按时间段组织的工作内容
4. 待跟进事项：从记录中提取的待办项
```
