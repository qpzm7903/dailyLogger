# Spec: AI-005 本地模型支持 (Ollama)

## 功能需求

支持本地部署的 Ollama 模型，允许用户在离线环境中使用 DailyLogger。

## 验收条件

### AC1: 支持配置 Ollama API 端点
- Given 用户打开设置界面
- When 配置 API Base URL 为 `http://localhost:11434/v1`
- Then 系统识别为 Ollama 端点并调整请求格式
- Given Ollama 端点配置
- When API Key 为空
- Then 系统正常工作（Ollama 不需要 API Key）

### AC2: 支持本地模型列表选择
- Given 用户配置了 Ollama 端点
- When 点击"获取模型列表"按钮
- Then 显示已安装的 Ollama 模型列表
- Given 模型列表加载成功
- When 用户选择某个模型
- Then 自动填充到模型名称字段

### AC3: 离线模式可用
- Given 用户配置了 Ollama 本地端点
- When 网络断开时
- Then 截图分析和日报生成功能正常工作
- Given Ollama 服务未运行
- When 用户尝试使用 AI 功能
- Then 显示清晰的错误提示："Ollama 服务未运行，请先启动 Ollama"

## 技术设计

### Ollama API 兼容性

Ollama 提供与 OpenAI 兼容的 API：

| 端点 | Ollama | OpenAI |
|-----|--------|--------|
| Chat Completions | `/v1/chat/completions` | `/v1/chat/completions` |
| Models List | `/api/tags` | `/v1/models` |
| 认证 | 无需或可选 | Bearer Token |

### 后端变更

**新增 Tauri Commands**:
| 命令 | 描述 |
|-----|------|
| `detect_ollama_endpoint` | 检测是否为 Ollama 端点 |
| `get_ollama_models` | 获取 Ollama 已安装模型列表 |
| `test_api_connection` | 测试 API 连接（支持 OpenAI 和 Ollama） |

**Settings 扩展**:
```sql
ALTER TABLE settings ADD COLUMN is_ollama INTEGER DEFAULT 0;
```

### Ollama 模型列表 API

```bash
# Ollama 原生 API
GET http://localhost:11434/api/tags
Response: {"models": [{"name": "llama3.2:latest", ...}]}

# OpenAI 兼容 API
GET http://localhost:11434/v1/models
Response: {"data": [{"id": "llama3.2:latest", ...}]}
```

### 端点检测逻辑

```rust
fn is_ollama_endpoint(url: &str) -> bool {
    url.contains("localhost:11434") ||
    url.contains("127.0.0.1:11434") ||
    // 支持自定义 Ollama 端口
    url.contains(":11434/v1")
}
```

### 错误处理增强

```rust
// Ollama 特定错误提示
match error_message {
    "connection refused" if is_ollama => "Ollama 服务未运行，请先启动 Ollama",
    _ => generic_error_message
}
```

## 风险

- Ollama Vision 模型支持有限，需要验证 llama3.2-vision 或 llava 模型
- 本地模型性能可能低于云端模型
- 大模型内存占用可能影响用户体验
