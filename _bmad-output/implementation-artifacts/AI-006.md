# Story 3.6: 自定义 API Headers

Status: ready-for-dev

## Story

As a DailyLogger 用户,
I want 自定义请求大模型 API 的 Headers,
so that 我可以使用各种 OpenAI 兼容的 API 服务（如 OpenRouter、Claude Code 等），这些服务需要特定的认证或标识 Headers。

## Acceptance Criteria

1. 用户可以在设置界面添加/编辑/删除自定义 Header（key-value 形式）
2. 系统预置常用 API 服务的 Header 模板（OpenRouter、Azure OpenAI、Claude Code 等）
3. 所有 LLM API 调用（截图分析、日报生成、周报/月报）自动携带自定义 Headers
4. Headers 存储在数据库中，敏感值可选择加密存储
5. 支持一键清除所有自定义 Headers
6. 提供 Header 预览功能，方便用户确认配置正确

## Dev Notes

### 预设 Header 模板

```json
[
  {
    "name": "OpenRouter",
    "headers": [
      {"key": "HTTP-Referer", "value": "https://dailylogger.app", "sensitive": false},
      {"key": "X-Title", "value": "DailyLogger", "sensitive": false}
    ]
  },
  {
    "name": "Azure OpenAI",
    "headers": [
      {"key": "api-key", "value": "", "sensitive": true}
    ],
    "note": "api-key header 替代 Authorization header"
  },
  {
    "name": "Claude Code",
    "headers": [
      {"key": "anthropic-version", "value": "2023-06-01", "sensitive": false}
    ]
  }
]
```

### API 调用 Header 合并逻辑

```rust
// 1. 先设置 Content-Type
let mut request = client.post(&endpoint)
    .header("Content-Type", "application/json")
    .json(&request_body);

// 2. 设置 Authorization（如果 api_key 非空且没有自定义替代）
if !config.api_key.is_empty() {
    // 检查自定义 headers 是否已包含 Authorization 或替代 header
    let has_auth = custom_headers.iter().any(|h|
        h.key.to_lowercase() == "authorization" ||
        h.key.to_lowercase() == "api-key"
    );
    if !has_auth {
        request = request.header("Authorization", format!("Bearer {}", config.api_key));
    }
}

// 3. 应用自定义 headers
for header in custom_headers {
    request = request.header(&header.key, &header.value);
}
```

- 遵循现有 Settings 字段命名规范：snake_case
- JSON 序列化使用 serde_json
