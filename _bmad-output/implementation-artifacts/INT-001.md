# Story 6.1: Notion 导出支持完善

Status: review

## Story

As a DailyLogger 用户,
I want 将完整报告内容导出到 Notion 数据库,
so that 我可以在 Notion 中查看和管理工作日报，而不仅仅是标题。

## 背景

Notion 导出功能已有基础实现（`src-tauri/src/notion.rs`），但当前实现仅创建一个只有标题的页面，忽略完整的报告内容。本故事需要完善此功能，将完整的 Markdown 报告内容写入 Notion 页面。

## Acceptance Criteria

1. 日报生成后，完整的 Markdown 内容被写入 Notion 页面，而不仅仅是标题
2. 支持 Notion Block API 格式转换（Markdown → Notion Blocks）
3. 大型报告正确分块（Notion API 单次请求限制 100 个 blocks）
4. 前端 UI 已存在，用户可配置 Notion API Key 和 Database ID
5. 连接测试功能正常工作（已实现 `test_notion_connection`）
6. 支持在 Notion 页面中正确显示：
   - 标题（H1-H3）
   - 有序/无序列表
   - 代码块
   - 引用块
   - 粗体/斜体文本

### Notion Block API 结构

```json
{
  "type": "paragraph",
  "paragraph": {
    "rich_text": [
      {
        "type": "text",
        "text": { "content": "Hello " },
        "annotations": { "bold": false }
      },
      {
        "type": "text",
        "text": { "content": "World" },
        "annotations": { "bold": true }
      }
    ]
  }
}
```

### Block 类型映射

| Markdown | Notion Block Type |
|----------|------------------|
| `# H1` | `heading_1` |
| `## H2` | `heading_2` |
| `### H3` | `heading_3` |
| `- item` | `bulleted_list_item` |
| `1. item` | `numbered_list_item` |
| ``` code ``` | `code` |
| `> quote` | `quote` |
| `**bold**` | rich_text annotations.bold = true |
| `*italic*` | rich_text annotations.italic = true |

### Notion API 限制

- 单次 `append_block_children` 请求最多 100 blocks
- 需要分批处理大型报告
- Rich text 单个元素最多 2000 字符
