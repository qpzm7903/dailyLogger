## Challenges and Growth Areas

### 1. Vue Component Tests Consistently Missing

**Problem:** Both DATA-008 and STAB-002 had Vue component tests listed as tasks but not completed:
- DATA-008 Task 5.1: "前端组件测试" mentioned but no StatisticsPanel.test.ts found during review
- STAB-002 Task 5.3: "Vue 组件测试" for auto backup not implemented

**Root Cause:** Frontend component testing seems to be deprioritized when stories run long.

**Lesson:** Vue component tests should be verified before marking stories done, or explicitly descoped with MEDIUM severity flag.

### 2. Markdown Export Incomplete

**Problem:** DATA-008 AC5 stated "CSV 或 Markdown" export but only CSV was implemented.

**Root Cause:** Story acceptance criteria had ambiguity - "CSV 或 Markdown" implies choice but developer interpreted as "CSV is sufficient."

**Lesson:** If an AC lists options with "ou/or", clarify which is required vs optional.

### 3. File List Accuracy

**Problem:** STAB-002 had a file discrepancy - `src-tauri/src/synthesis/mod.rs` was modified but not listed in the story's File List.

**Root Cause:** During implementation, additional changes were made to files not anticipated during planning.

**Lesson:** Dev Agent should verify File List matches actual git changes before marking done.

## Key Insights

### Technical Insights

1. **Database Migration Pattern is Solid**
   - ALTER TABLE with DEFAULT values works reliably
   - No data loss incidents in Epic 11
   - Settings table extension pattern is now well-established

2. **Statistics Infrastructure is Reusable**
   - DATA-008 created Statistics struct, DateRange, DailyStatistic
   - This infrastructure can be extended for future analytics features
   - Daily breakdown pattern enables trend visualization

3. **Auto Backup Architecture is Well-Designed**
   - tokio-based scheduler is non-blocking
   - auto- prefix for file naming is a simple but effective separation
   - Retention policy with configurable bounds (3-20) is user-friendly

4. **i18n Pattern Propagation**
   - DATA-007 extended i18n to export functionality
   - Consistent use of locales/zh-CN.json and locales/en.json
   - 7 languages now supported across the application

### Process Insights

1. **Dev Checklist Items from Epic 10 Were Never Added**
   - The action items to add verification steps to Dev Checklist were not implemented
   - This contributed to issues like composable integration (DATA-008) and file list accuracy (STAB-002)
   - **Recommendation:** Add these to the story development workflow explicitly

2. **Task Completion vs Story Completion**
   - Stories are marked "done" even when some tasks are incomplete
   - This creates confusion about actual implementation state
   - DATA-008 story status was "review" but tasks showed unchecked items

3. **Cross-Epic Action Items Need Own Tracking**
   - Action items from Epic 10's retrospective were not carried forward as sprint tasks
   - They exist in the retrospective doc but don't appear in sprint-status.yaml
   - **Recommendation:** Add epic retrospective action items to sprint-status.yaml
