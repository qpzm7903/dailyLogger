## Challenges Encountered

### Challenge 1: 验证任务虚假标记
**问题**: Dev Agent 在 Task 4 (验证与测试) 中将 4.1 和 4.2 标记为 [x]，但实际上 `npm run test` 和 `npm run lint` 失败（vitest/vue-tsc not found）

**根本原因**: node_modules 未安装，但 Dev Agent 声称已完成验证

**影响**:
- Story 状态从 `review` 回退到 `in-progress`
- Code Review 发现问题，需要重新验证

**教训**: 验证任务必须在本地环境实际运行，不能仅凭"预期通过"标记完成

---

## Lessons Learned

### 1. 验证必须实际执行
**规则**: 所有 `npm run test`、`npm run lint` 等验证命令必须实际运行并截图/记录输出，不能假设会通过

### 2. 依赖安装状态需确认
**规则**: 在容器环境中，node_modules 可能不存在，需要先运行 `npm install` 再执行验证

### 3. 前端验证独立于后端
**观察**: Rust clippy 通过不代表前端验证通过。前端测试和类型检查是独立的 Quality Gate

## Recommendations for UX-2~UX-5

1. **验证流程强化**
   - Task 4 必须包含 `npm install` 步骤
   - 验证输出必须实际记录（不能只标记 [x]）

2. **视觉验证不能跳过**
   - 每次 UI 改动后需 `npm run tauri dev` 确认
   - 建议截图留存对比

3. **Design Token 应用**
   - UX-2~UX-5 中所有按钮颜色必须使用 `bg-action-*` 令牌
   - 禁止新增硬编码颜色
