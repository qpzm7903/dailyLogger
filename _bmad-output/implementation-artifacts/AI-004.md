# Story 3.4: 工作分类标签生成

Status: done

## Story

作为一个 DailyLogger 用户，
我希望 AI 能自动为每条记录生成工作分类标签，
以便我可以快速了解工作内容分布，并按类别筛选查看记录。

## Acceptance Criteria

### AC1 - 每条记录自动分类

- Given 用户启动自动捕获
- When AI 分析截图完成
- Then 记录自动附加分类标签（如：开发、会议、写作、学习等）
- Given 用户查看记录列表
- When 记录有标签
- Then 显示标签徽章（彩色小标签样式）

### AC2 - 支持自定义标签体系

- Given 用户打开设置界面
- When 查看"标签分类"部分
- Then 可编辑自定义标签列表（textarea，每行一个标签）
- Given 用户保存自定义标签
- When 下次 AI 分析时
- Then AI 从自定义标签体系中选择最匹配的标签
- Given 用户清空自定义标签
- When 下次 AI 分析时
- Then AI 使用默认标签分类

### AC3 - 可按标签筛选记录

- Given 今日有多条记录且带有不同标签
- When 用户点击某个标签
- Then 记录列表仅显示包含该标签的记录
- Given 用户点击"全部"或清除筛选
- Then 显示所有记录
- Given 用户查看标签筛选器
- Then 显示所有已使用的标签及其记录数量

#[test]
fn screen_analysis_parses_tags_field() {
    let json = r#"{"current_focus": "测试", "active_software": "App", "context_keywords": [], "tags": ["开发", "测试"]}"#;
