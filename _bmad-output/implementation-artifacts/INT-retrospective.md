## 经验教训 (Lessons Learned)

### 1. 验证型 Story 的价值
对于已有代码基础的功能，验证型 Story 比开发型更高效。先确认现有实现再补充缺失，避免重复工作。

### 2. 第三方库 API 验证
第三方库的 API 可能与文档不同步，建议在开始实现前先编写简单的 POC 测试验证。

### 3. API 限制处理策略
外部 API 通常有请求限制，提前了解并实现分块逻辑很重要。分块逻辑应该模块化以便复用。

### 4. 累积器模式
对于需要状态追踪的转换逻辑（如 Markdown 行内格式），累积器模式是一个清晰的解决方案。

### 5. 错误隔离设计
第三方服务调用失败不应影响核心功能，使用 `Option<bool>` 返回值处理"未配置"和"发送结果"两种状态。

### 6. 本地格式检查的重要性
提交前必须运行 `cargo fmt -- --check` 和 `cargo clippy -- -D warnings`，节省 CI 时间。

### 7. 类型定义同步更新
前后端类型定义同步创建避免类型不匹配，TypeScript 类型安全提升开发体验。

### 8. 状态处理的重要性
加载、错误、未配置三种状态缺一不可，良好的状态处理提升用户体验。
