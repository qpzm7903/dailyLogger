## 挑战 (Challenges)

### 1. 实时更新逻辑遗漏 (EXP-001)
**问题**: TimelineWidget 暴露了 refresh() 方法，但 Dashboard 未调用
**发现**: Code Review 阶段
**解决**: 添加 watch 监听 todayRecords 变化并调用 refresh()
**教训**: 新组件的事件绑定需要集成测试验证

### 2. SQL NULL 处理 (EXP-005)
**问题**: SUM() 对空数据集返回 NULL，导致前端显示异常
**发现**: 单元测试阶段
**解决**: 使用 COALESCE(SUM(...), 0) 处理
**教训**: 数据库查询要考虑边界情况

### 3. 事件冒泡处理 (EXP-003)
**问题**: 重分析按钮点击触发卡片点击事件
**解决**: 使用 @click.stop 阻止冒泡

---

## 经验教训 (Lessons Learned)

### 1. 新组件生命周期验证
组件暴露的方法（如 refresh()）必须在父组件中正确调用。建议在开发时添加集成测试验证，而非仅在 Code Review 中发现。

### 2. SQL 边界情况处理
聚合函数（SUM、AVG、COUNT）对空数据集的行为需要明确。使用 COALESCE 是标准解决方案，应在初始实现时就考虑。

### 3. 验证型 Story 的模式固化
对于已存在代码的功能，验证优先。后端已实现时，前端补全即可。这种模式在 EXP-003、EXP-004 中再次验证有效。

### 4. 前端状态独立管理
多条记录的加载状态应使用 Set 独立管理，避免相互影响（EXP-003 的 reanalyzingIds 模式）。

### 5. 防抖与搜索体验
300ms 防抖对于搜索场景是合适的平衡，既避免频繁请求，又保持响应速度。
