# Story 3.1: 多模型支持配置

Status: done

## Story

作为一个 DailyLogger 用户，
我希望能够为截图分析和日报生成配置不同的 AI 模型，
以便在保证分析精度的同时降低 API 成本，同时能够验证配置是否正确并了解模型的能力限制。

## Acceptance Criteria

### AC1 - 分析模型和总结模型可分别配置
- Given 用户打开设置界面
- When 查看"截图分析 (Vision)"和"日报生成"部分
- Then 显示两个独立的模型配置字段
- Given 用户为分析配置 gpt-4o，为日报配置 gpt-4o-mini
- When 保存设置并执行相应操作
- Then 截图分析使用 gpt-4o，日报生成使用 gpt-4o-mini

### AC2 - 支持测试连接验证
- Given 用户已配置 API Base URL、API Key 和模型名称
- When 点击"测试连接"按钮
- Then 发送一个简单的 API 请求验证配置
- And 显示测试结果（成功/失败及原因）
- Given 配置无效
- When 点击测试连接
- Then 显示具体的错误信息（如：API Key 无效、模型不支持等）

### AC3 - 显示模型支持的上下文窗口
- Given 用户已配置有效 API 并测试连接成功
- When 点击"查看模型信息"按钮
- Then 调用 API 获取模型详情
- And 显示模型的上下文窗口大小（如：128K tokens）
- Given API 不支持模型信息查询
- Then 显示"无法获取，请参考模型文档"

### 测试连接实现方案

```rust
// memory_storage/mod.rs 或新文件
#[derive(Serialize)]
pub struct ConnectionTestResult {
    pub success: bool,
    pub message: String,
    pub latency_ms: Option<u64>,
}

#[command]
pub async fn test_api_connection(
    api_base_url: String,
    api_key: String,
    model_name: String,
) -> Result<ConnectionTestResult, String> {
    let client = reqwest::Client::new();
    let start = std::time::Instant::now();

    // 发送最简单的请求
    let request_body = serde_json::json!({
        "model": model_name,
        "messages": [{"role": "user", "content": "Say 'ok'"}],
        "max_tokens": 5
    });

    let response = client
        .post(&format!("{}/chat/completions", api_base_url))
        .header("Authorization", format!("Bearer {}", api_key))
        .json(&request_body)
        .send()
        .await;

    match response {
        Ok(resp) if resp.status().is_success() => {
            Ok(ConnectionTestResult {
                success: true,
                message: "连接成功".to_string(),
                latency_ms: Some(start.elapsed().as_millis() as u64),
            })
        }
        Ok(resp) => {
            let status = resp.status();
            let body = resp.text().await.unwrap_or_default();
            Ok(ConnectionTestResult {
                success: false,
                message: format!("API 错误 ({}): {}", status, body),
                latency_ms: Some(start.elapsed().as_millis() as u64),
            })
        }
        Err(e) => {
            Ok(ConnectionTestResult {
                success: false,
                message: format!("连接失败: {}", e),
                latency_ms: None,
            })
        }
    }
}
```

### 模型信息查询实现方案

```rust
#[derive(Serialize)]
pub struct ModelInfo {
    pub model_id: String,
    pub context_window: Option<u64>,
    pub error: Option<String>,
}

#[command]
pub async fn get_model_info(
    api_base_url: String,
    api_key: String,
    model_name: String,
) -> Result<ModelInfo, String> {
    let client = reqwest::Client::new();

    // OpenAI 兼容 API 的 /models 端点
    let response = client
        .get(&format!("{}/models/{}", api_base_url, model_name))
        .header("Authorization", format!("Bearer {}", api_key))
        .send()
        .await;

    match response {
        Ok(resp) if resp.status().is_success() => {
            let json: serde_json::Value = resp.json().await.unwrap_or_default();
            // OpenAI 返回格式: {"id": "gpt-4o", "context_window": 128000}
            // 或在某些 API 中是 max_tokens
            let context_window = json.get("context_window")
                .or_else(|| json.get("max_tokens"))
                .and_then(|v| v.as_u64());

            Ok(ModelInfo {
                model_id: model_name,
                context_window,
                error: None,
            })
        }
        Ok(resp) => {
            Ok(ModelInfo {
                model_id: model_name,
                context_window: None,
                error: Some(format!("无法获取模型信息 (状态: {})", resp.status())),
            })
        }
        Err(e) => {
            Ok(ModelInfo {
                model_id: model_name,
                context_window: None,
                error: Some(format!("请求失败: {}", e)),
            })
        }
    }
}
```
