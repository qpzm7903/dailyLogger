## Lessons Learned

### Technical

1. **ErrorBoundary 必须集成到 App.vue**
   - 问题：初始实现 ErrorBoundary.vue 但未集成到 App.vue，代码审查发现此问题
   - 解决：修改 App.vue 用 `<ErrorBoundary>` 包裹主内容
   - 建议：组件实现后应立即验证是否正确集成到父组件

2. **非阻塞 UI 组件的 pointer-events 处理**
   - 问题：ErrorToast 的 `.fixed` div 使用 `z-50` 阻挡了 Save 按钮点击
   - 解决：容器使用 `z-40` + `pointer-events-none`，dismiss 按钮单独设置 `pointer-events-auto`
   - 建议：fixed/absolute 定位的错误提示组件必须测试与下方按钮的交互

3. **i18n 翻译必须与功能同步**
   - 问题：ErrorBoundary 使用了 `errorBoundary.title` 但初始翻译未添加
   - 解决：同步更新 zh-CN.json 和 en.json
   - 建议：创建涉及文本的组件时，同时准备翻译条目

### Process

- **代码审查发现 HIGH 问题**：ErrorBoundary 未集成是 AC 未完全实现的标志，说明 self-review 不足
- **E2E 测试捕获 UI 问题**：Playwright 测试发现了 ErrorToast 阻挡交互的问题，自动化测试价值体现
- **多轮修复完成**：从 code review 到 E2E fix，经历 2 次 commit 才完全 pass
