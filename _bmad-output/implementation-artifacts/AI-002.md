# Story 3.2: 自定义分析 Prompt

Status: ready-for-dev

## Story

作为一个 DailyLogger 用户，
我希望能够自定义截图分析的 Prompt，
以便根据我的工作场景调整 AI 分析的侧重点和输出格式，并在需要时可以快速恢复默认设置。

## Acceptance Criteria

### AC1 - 设置界面支持编辑 analysis_prompt
- Given 用户打开设置模态框
- When 查看"截图分析 (Vision)"部分
- Then 显示"分析 Prompt"多行文本框
- Given 用户编辑了分析 Prompt
- When 点击保存
- Then 新的 Prompt 被保存到数据库
- And 后续截图分析使用新 Prompt

**当前状态**: ✅ 已实现 - SettingsModal.vue 已有 analysis_prompt textarea

### AC2 - 提供默认 Prompt 模板
- Given 用户首次使用或未设置过 analysis_prompt
- When 进行截图分析时
- Then 使用内置的 DEFAULT_ANALYSIS_PROMPT
- Given 用户在设置界面查看
- When 分析 Prompt 输入框为空
- Then 显示 placeholder 提示"留空使用默认 Prompt"

**当前状态**: ✅ 已实现 - auto_perception/mod.rs 定义了 DEFAULT_ANALYSIS_PROMPT，analyze_screen 函数正确处理空值

### AC3 - 支持重置为默认值
- Given 用户设置了自定义 analysis_prompt
- When 点击"重置为默认"按钮
- Then analysis_prompt 字段清空
- And 显示 toast 提示"已重置为默认 Prompt"
- Given 用户重置后进行截图分析
- When AI 分析执行
- Then 使用 DEFAULT_ANALYSIS_PROMPT

**当前状态**: ❌ 未实现 - 需要添加重置按钮

const DEFAULT_ANALYSIS_PROMPT: &str = r#"Analyze this screenshot and return a JSON object with:
