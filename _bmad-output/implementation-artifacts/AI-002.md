# Story 3.2: 自定义分析 Prompt

Status: done

## Story

作为一个 DailyLogger 用户，
我希望能够自定义截图分析的 Prompt 模板，
以便让 AI 分析更符合我的工作场景和偏好，同时可以随时重置为默认值。

## Acceptance Criteria

### AC1 - 设置界面支持编辑 analysis_prompt
- Given 用户打开设置界面
- When 查看"截图分析 (Vision)"部分
- Then 显示"分析 Prompt"文本框，可编辑自定义 Prompt
- Given 用户输入自定义 Prompt 并保存
- When 下次截图分析时
- Then AI 使用自定义 Prompt 进行分析

### AC2 - 提供默认 Prompt 模板
- Given 用户打开设置界面
- When 将鼠标悬停在"分析 Prompt"字段旁的"查看默认"按钮上
- Then 显示默认 Prompt 的预览（tooltip 或弹窗）
- Given 用户点击"查看默认"按钮
- Then 显示完整的默认 Prompt 内容

### AC3 - 支持重置为默认值
- Given 用户已输入自定义 Prompt
- When 点击"重置为默认"按钮
- Then 清空 analysis_prompt 字段，系统将使用默认 Prompt
- Given 用户清空 Prompt 后保存
- When 下次截图分析时
- Then AI 使用默认 Prompt 进行分析

### UI 设计建议

在"分析 Prompt"字段下方添加两个按钮：

```html
<div class="flex gap-2 mt-2">
  <button @click="showDefaultPrompt" class="text-xs text-gray-400 hover:text-primary">
    查看默认
  </button>
  <button @click="resetPrompt" class="text-xs text-gray-400 hover:text-primary">
    重置为默认
  </button>
</div>
```

默认 Prompt 显示方式选项：
1. **Modal 弹窗**：复用现有 modal 模式
2. **Collapsible 展开**：在 textarea 下方可折叠显示
3. **Tooltip**：鼠标悬停显示（内容较长不适合）

推荐使用 **Modal 弹窗**，因为默认 Prompt 内容较长。
