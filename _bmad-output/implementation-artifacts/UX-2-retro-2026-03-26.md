## Lessons Learned

### 1. 依赖文件变更需记录
**规则**: package-lock.json、Cargo.lock 等依赖锁定文件发生变更时，必须记录到 Dev Notes File List

### 2. additionalOptions 模式值得推广
**观察**: ReportDropdown 的 additionalOptions prop 实现了"常驻操作 + 可扩展操作"的分离设计
**适用场景**: 任何有"主操作 + 次要/高级操作"的 UI 场景
**示例**: 设置菜单、文件操作右键菜单、工具栏

### 3. emit 设计应区分操作类型
**观察**: UX-2 区分了 `generate`（报告生成）和 `openModal`/`customAction`（其他操作）
**好处**: 组件职责清晰，父组件可以针对不同操作类型做不同处理
