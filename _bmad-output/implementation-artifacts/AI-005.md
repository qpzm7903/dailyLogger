# Story AI-005: 本地模型支持 (Ollama)

Status: done

## Story

As a 用户在没有网络连接的环境下工作,
I want 能够使用本地部署的 Ollama 模型进行截图分析和日报生成,
so that 即使在离线环境下也能正常使用 DailyLogger 的核心功能.

## Acceptance Criteria

1. **AC1: 支持配置 Ollama API 端点**
   - Given 用户打开设置界面
   - When 配置 API Base URL 为 `http://localhost:11434/v1`
   - Then 系统识别为 Ollama 端点并调整请求格式
   - Given Ollama 端点配置且 API Key 为空
   - When 用户使用 AI 功能
   - Then 系统正常工作（Ollama 不需要 API Key）

2. **AC2: 支持本地模型列表选择**
   - Given 用户配置了 Ollama 端点
   - When 点击"获取模型列表"按钮
   - Then 显示已安装的 Ollama 模型列表
   - Given 模型列表加载成功
   - When 用户选择某个模型
   - Then 自动填充到模型名称字段

3. **AC3: 离线模式可用**
   - Given 用户配置了 Ollama 本地端点
   - When 网络断开时
   - Then 截图分析和日报生成功能正常工作
   - Given Ollama 服务未运行
   - When 用户尝试使用 AI 功能
   - Then 显示清晰的错误提示："Ollama 服务未运行，请先启动 Ollama"

### Ollama API 端点

| 功能 | OpenAI | Ollama 原生 | Ollama 兼容 |
|-----|--------|------------|-------------|
| Chat | `/v1/chat/completions` | `/api/chat` | `/v1/chat/completions` |
| Models | `/v1/models` | `/api/tags` | `/v1/models` |
| 认证 | Bearer Token | 无 | 无（或可选） |

**推荐使用 Ollama 的 OpenAI 兼容 API** (`/v1/*` 端点)，与现有代码改动最小。

### 检测 Ollama 端点

```rust
fn is_ollama_endpoint(url: &str) -> bool {
    let url_lower = url.to_lowercase();
    url_lower.contains("localhost:11434") ||
    url_lower.contains("127.0.0.1:11434") ||
    url_lower.contains(":11434/v1")
}
```

### 获取 Ollama 模型列表

```rust
// Ollama 原生 API (推荐)
async fn get_ollama_models(base_url: &str) -> Result<Vec<String>, String> {
    let url = format!("{}/api/tags", base_url.trim_end_matches("/v1"));
    let resp = reqwest::get(&url).await.map_err(|e| e.to_string())?;
    let json: serde_json::Value = resp.json().await.map_err(|e| e.to_string())?;
    let models = json["models"]
        .as_array()
        .map(|arr| arr.iter().filter_map(|m| m["name"].as_str().map(String::from)).collect())
        .unwrap_or_default();
    Ok(models)
}
```

### 错误处理增强

```rust
// 区分 Ollama 和 OpenAI 错误
fn format_api_error(error: &str, is_ollama: bool) -> String {
    if error.contains("connection refused") && is_ollama {
        "Ollama 服务未运行，请先启动 Ollama (ollama serve)".to_string()
    } else if error.contains("connection refused") {
        "无法连接到 API 服务器，请检查网络和 Base URL".to_string()
    } else {
        error.to_string()
    }
}
```

### Vision 模型支持

Ollama 支持 Vision 的模型：
- `llava` - 推荐，视觉理解能力强
- `llama3.2-vision` - 最新，但需要较大内存
- `bakllava` - 另一个选择

用户需要使用 `ollama pull llava` 安装。
