## Code Review Issue - HIGH Severity

### Issue: Tooltip Implementation is Broken

**位置**: `src/components/layout/Sidebar.vue`

**问题描述**:
1. 按钮元素没有 `group` class
2. Tooltip span 是 button 的兄弟元素，不是子元素
3. `group-hover:opacity-100` CSS 模式要求父元素有 `group` class 且 tooltip 是子元素

**代码问题**:
```vue
<!-- Broken: tooltip 是兄弟元素，且父元素没有 group class -->
<button ...>
  <component :is="item.icon" ... />
  <span v-if="!isCollapsed" ...>{{ item.label }}</span>
  <!-- Tooltip 是 SIBLING，不是 child - group-hover 不起作用! -->
  <span v-if="isCollapsed" class="absolute left-14 ... group-hover:opacity-100 ...">
    {{ item.label }}
  </span>
</button>
```

### Fix Applied

**方案**: 采用 Option A - 移除损坏的 CSS tooltip spans，依赖原生浏览器 `:title` 属性

**变更**:
- 移除了 nav items 的 tooltip spans (原 lines 37-43)
- 移除了 settings 按钮的 tooltip span (原 lines 61-66)
- 保留了 `:title="isCollapsed ? item.label : undefined"` 属性

## Lessons Learned

### 1. CSS `group-hover` 模式需要正确结构
**规则**: 使用 `group-hover:` Tailwind 变体时，父元素必须：
- 有 `group` class
- 子元素才有 `group-hover:` 效果

**正确模式**:
```vue
<div class="group relative">
  <button>...</button>
  <span class="absolute ... opacity-0 group-hover:opacity-100">Tooltip</span>
</div>
```

### 2. 原生 `:title` 属性是可靠的 tooltip 方案
**场景**: 简单的 hover tooltip 需求
**优点**: 无需额外 CSS/JS，浏览器原生支持
**限制**: 样式不灵活，无法自定义外观
