# Epic 9: 视觉与交互体验系统化升级 (UX-REDESIGN)

**状态**: backlog
**优先级**: P1
**版本**: v3.0.0
**创建日期**: 2026-03-22

---

## Executive Summary

DailyLogger 当前已有良好的深色 glassmorphism 骨架（Tailwind v4 + TailwindCSS），但随着功能不断叠加，出现了明显的视觉混乱问题。本 Epic 系统性地诊断并规划 UI/UX 改进，建立设计令牌体系、统一组件视觉规范、优化信息架构，目标是**让工具消失，让工作流浮现**。

**触发点**：Sprint 8 功能已落地（Session Manager + 批量分析），项目进入成熟期，用户体验欠债需要偿还。

## 一、现状分析（UX 诊断）

### 1.1 色彩系统问题

| 元素 | 当前值 | 问题 |
|------|--------|------|
| 背景底层 | `#0f172a` (darker) | ✅ 正常 |
| 卡片层 | `#1e293b` (dark) | ✅ 正常 |
| 主色 | `#3b82f6` (blue-500) | ✅ 正常 |
| 功能按钮 | orange / teal / purple / indigo / green | ❌ **无语义，随意使用** |

**核心问题**：
- 仅有 4 个设计令牌，按钮颜色完全靠硬编码
- "自定义报告"用橙色、"对比分析"用蓝绿、"按日期重新分析"用紫色、"重新分析今天"用靛蓝——没有任何语义逻辑
- 新功能添加时无颜色选择标准，导致色彩混乱加剧

### 1.2 布局问题

1. **仪表板操作按钮过载**
   - `今日工作流` 区域右侧塞了 5 个按钮（ReportDropdown + 自定义报告 + 对比分析 + 按日期重新分析 + 重新分析今天）
   - 严重挤压视觉空间，首次使用者根本不知道从哪里开始
   - 按钮层级不清晰：主操作（开始录制）和辅助操作（查看历史）视觉权重相同

2. **侧边栏可发现性低**
   - 仅有 emoji（🗒️📚🔍🏷️📤📈💾），无文字标签
   - 导航可发现性依赖 hover tooltip，在触屏或截图预览场景完全失效
   - 没有激活状态指示器，用户不知道当前在哪个视图

3. **滚动嵌套问题**
   - 记录列表固定 `max-h-80` 在卡片内滚动，内外页面也在滚动
   - 造成双层滚动体验割裂，交互体验割裂感强

4. **Header 资源浪费**
   - 顶部仅显示 "DailyLogger" 文字 + 时钟，大量空间未使用
   - 无实时状态反馈

5. **输出文件卡片视觉噪音**
   - 垂直平铺日报/周报/月报/自定义/对比
   - 未生成时显示多个"尚未生成"空状态，视觉噪音大

6. **图标系统局限**
   - 全部使用 emoji，不同平台渲染差异大
   - 无法精细控制样式（颜色、尺寸、动效）
   - 复制粘贴时产生意外符号

### 1.3 交互问题

- 报告生成类操作散落在多处（ReportDropdown + 4 个独立按钮），应合并为统一入口
- 无键盘导航支持的可见提示
- 弹窗无 ESC 键关闭提示

## 二、设计方向

### 核心理念
> **"让工具消失，让工作流浮现"** — 减少 UI 元素的存在感，强化内容和数据的呈现。

### 色彩升级方向
从硬编码颜色迁移到**语义化令牌体系**：

- `--color-action-primary`: 蓝色（主操作）
- `--color-action-secondary`: 灰色（辅助操作）
- `--color-action-danger`: 红色（破坏性操作）
- `--color-action-neutral`: 深灰（工具操作）
- `--color-status-success/warning/error/info`: 状态色

### 布局升级方向

- 侧边栏改为可折叠式：展开时显示图标+文字，折叠时只显示图标，加入激活状态高亮
- 将报告操作收纳为单一"生成报告"按钮 + 下拉面板，隐藏低频高级操作
- 记录列表改为全高度视图（去掉 max-h 限制）或分页（每页 20 条）
- Header 加入今日状态摘要（记录数、自动捕获状态）
- 输出文件卡片改为 Tab 式（日报/周报/月报），统一空状态展示

## 八、设计决策（已确认）

### 决策 1: 图标系统
| 方案 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| 继续用 emoji | 轻量、无依赖 | 跨平台渲染差、无样式控制 | ❌ |
| Lucide Vue Next | 精美、轻量、Vue 3 友好 | +2.5KB 依赖 | ✅ |
| Material Icons | 功能全 | 包体积大 | ❌ |

**结论**: ✅ 引入 `lucide-vue-next`，替换 emoji 导航图标

### 决策 2: 侧边栏折叠偏好持久化
| 方案 | 优点 | 缺点 | 决策 |
|------|------|------|------|
| localStorage 持久化 | UX 连贯 | 复杂度增加、需考虑响应式 | ❌ |
| Vue ref 本地状态 | 简单、快速 | 刷新重置 | ✅ |

**结论**: ❌ 不持久化，仅用 Vue `ref` 管理本地状态（刷新重置）

### 决策 3: 报告按钮合并策略
| 方案 | 保留独立 | 收入面板 | 决策 |
|------|---------|---------|------|
| 全部独立 | 全部 | 无 | ❌ 视觉混乱 |
| 部分合并 | 日报/周报/月报 | 自定义、对比、按日期、重新分析 | ✅ |
| 全部合并 | 无 | 全部 | ❌ 低频操作也难找 |

**结论**: ✅ 保留最常用快捷操作（日报/周报/月报），其余收入 ReportDropdown 的下拉/折叠面板

## 九、风险与缓解

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 颜色令牌定义不完整 | UX-2 ~ UX-5 无法开始 | UX-1 有专项 AC 验证 |
| 按钮 CSS 冲突 | 样式混乱 | 使用 Tailwind @apply，避免全局冲突 |
| 图标引入造成 bundle 增长 | 性能下降 | lucide-vue-next 按需导入，<5KB |
| 侧边栏可折叠在移动端体验差 | 小屏幕无法正常显示 | 响应式检测，小屏自动折叠 |
| 仪表板重组导致用户迷失 | 学习成本增加 | 灰度发布，A/B 测试 |
