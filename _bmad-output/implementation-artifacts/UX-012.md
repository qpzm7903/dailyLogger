## Overview

Implement virtual scrolling for HistoryViewer component's record list. When history records exceed 200 entries, only render items within the current viewport to avoid performance degradation and high memory usage from rendering thousands of DOM nodes.

## Current Implementation State

The virtual scrolling infrastructure is already in place:
- `@tanstack/vue-virtual` dependency installed
- `HistoryViewer.vue` has virtual scroll configuration and rendering logic (lines 68-115, 209-245)
- `VIRTUAL_SCROLL_CONFIG` with threshold: 100, itemHeight: 80, overscan: 5
- `shouldUseVirtualScroll` computed enables virtual scrolling when records > threshold
- Virtualizer is initialized with `useVirtualizer`

### Virtual Scroll Configuration

```typescript
const VIRTUAL_SCROLL_CONFIG = {
  itemHeight: 80,     // Fixed height per record (px)
  overscan: 5,        // Render extra items outside viewport
  threshold: 100,     // Enable virtual scroll above this count
}
```

### Implementation Pattern

```typescript
const shouldUseVirtualScroll = computed(() => records.value.length > VIRTUAL_SCROLL_CONFIG.threshold)

const virtualizer = useVirtualizer({
  count: records.value.length,
  getScrollElement: () => scrollContainer.value,
  estimateSize: () => VIRTUAL_SCROLL_CONFIG.itemHeight,
  overscan: VIRTUAL_SCROLL_CONFIG.overscan,
})

const virtualItems = computed(() => virtualizer.getVirtualItems())
```
