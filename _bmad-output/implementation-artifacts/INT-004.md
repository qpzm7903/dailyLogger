# Story 6.4: Slack/钉钉通知

Status: review

## Story

As a DailyLogger 用户,
I want 在报告生成后自动发送通知到 Slack 或钉钉群,
so that 我可以及时分享工作成果给团队成员，无需手动复制粘贴。

## Dev Notes

### 钉钉 Webhook API 参考

钉钉机器人使用 Incoming Webhook，支持 Markdown 格式：

```json
POST https://oapi.dingtalk.com/robot/send?access_token=XXX
Content-Type: application/json

{
  "msgtype": "markdown",
  "markdown": {
    "title": "工作日报 - 2026-03-21",
    "text": "### 工作日报\n\n- 完成功能开发\n- 修复 Bug"
  }
}
```

**响应格式**：
```json
{
  "errcode": 0,
  "errmsg": "ok"
}
```

### 报告生成中集成通知

```rust
// synthesis/mod.rs
use crate::slack;
use crate::dingtalk;

pub async fn generate_daily_summary() -> Result<String, String> {
    // ... 现有报告生成逻辑 ...

    // 发送通知（失败不影响主流程）
    let settings = memory_storage::get_settings_sync()?;
    let title = format!("工作日报 - {}", date);

    if slack::is_slack_configured(&settings) {
        if let Some(success) = slack::send_to_slack(&settings, &title, &content).await {
            if success {
                tracing::info!("Daily report sent to Slack");
            } else {
                tracing::warn!("Failed to send to Slack");
            }
        }
    }

    if dingtalk::is_dingtalk_configured(&settings) {
        if let Some(success) = dingtalk::send_to_dingtalk(&settings, &title, &content).await {
            if success {
                tracing::info!("Daily report sent to DingTalk");
            } else {
                tracing::warn!("Failed to send to DingTalk");
            }
        }
    }

    Ok(report_path)
}
```

### 注意事项

1. **错误隔离**：通知发送失败不应影响报告生成主流程，使用 `tracing::warn!` 记录错误
2. **异步处理**：通知发送是异步操作，使用 `tokio::spawn` 避免阻塞
3. **内容格式**：
   - Slack 使用 mrkdwn 格式
   - 钉钉使用 Markdown 格式（语法略有不同）
4. **内容截断**：Slack 有 4000 字符限制，钉钉有类似限制，需要截断处理
5. **测试运行**：使用 `cargo test --no-default-features` 避免 xcap 依赖
