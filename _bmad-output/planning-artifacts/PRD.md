# 产品需求文档 (PRD)

## DailyLogger - AI 工作流记忆与日报生成应用

---

### 1. 执行摘要

**产品名称**: DailyLogger
**版本**: 2.0.0
**文档状态**: 已批准
**最后更新**: 2026-03-22

DailyLogger 是一款桌面应用，通过自动截屏分析和手动速记相结合的方式，帮助用户记录工作上下文，并在一天结束时生成结构化日报，无缝输出到 Obsidian 知识库。

---

### 2. 产品愿景

成为知识工作者的"第二大脑"，无感记录工作流，让用户专注于创造性工作而非文档整理。

---

### 2.1 功能准入标准

每个新功能在进入 Epic 规划前，必须通过以下问题：

1. **直接提升记录体验**：这个功能是否让用户更准确/更高效地记录工作？
2. **使用频率**：用户每天会用到吗？（核心功能）还是偶尔用（辅助功能）？
3. **替代方案**：用户不用这个功能，会损失什么？损失小则低优先级。

**优先级矩阵**：

| 级别 | 使用场景 | 决策 |
|------|---------|------|
| P0 | 每次记录工作时都需要 | 必须做 |
| P1 | 每天使用 | 重要 |
| P2 | 每周使用 | 有价值 |
| P3 | 偶尔使用 | 待评估 |

> 集成类功能（IM 通知、第三方平台导出）通常属于 P2-P3，在 P0/P1 核心体验未达到卓越之前不优先。

### 3. 目标用户

| 用户类型 | 特征 | 核心需求 |
|---------|------|---------|
| 软件开发者 | 长时间编码，需要记录技术决策 | 自动记录编码上下文，生成技术日报 |
| 研究人员 | 阅读大量文献，需要整理笔记 | 截图分析论文内容，自动归类 |
| 产品经理 | 多任务切换，会议频繁 | 快速记录想法，自动生成会议纪要 |
| 内容创作者 | 需要素材积累和灵感整理 | 自动收集参考素材，整理创意日志 |

---

### 4. 成功指标

| 指标 | 目标值 | 测量方式 |
|-----|-------|---------|
| 日活跃用户 (DAU) | >60% 周活 | 应用启动次数 |
| 平均每日记录数 | >10 条/用户 | 数据库记录统计 |
| 日报生成率 | >80% | 生成次数/活跃天数 |
| 用户留存率 (7 日) | >50% | 回访用户比例 |
| NPS 净推荐值 | >40 | 用户调研 |

### 5. 用户旅程

#### 5.1 新用户入职流程

```
下载安装 → 首次启动 → 配置 API → 设置 Obsidian 路径 → 开始使用
```

**关键触点**:
- 首次启动时引导配置 API Base URL 和 API Key
- 提供测试连接功能验证配置
- 默认截图间隔 5 分钟，用户可随时调整

#### 5.2 日常使用流程

```
启动应用 (或托盘唤醒)
├─ 自动感知运行中 (后台截图分析)
├─ 随时按 Alt+Space 记录想法
├─ 工作间隙查看今日记录
└─ 下班时点击"生成日报" → 输出到 Obsidian
```

#### 5.3 日报生成流程

```
点击生成 → 汇总今日记录 → AI 分析整理 → Markdown 输出 → Obsidian 保存
```

### 6. 功能需求

#### 6.1 自动感知 (Auto-Perception)

**需求描述**: 定时截取屏幕并按工作时段组织，通过批量上下文分析提供准确的工作记录

**功能规格**:
- 可配置截图间隔（默认 5 分钟）
- 截图捕获后保存到本地，不立即调用 AI
- 系统自动检测工作时段（截图间隔 > 30 分钟 = 新时段，间隔可配置）
- 时段结束后自动触发批量分析（将该时段所有截图 + 上一时段上下文一起发送 AI）
- 支持用户手动触发指定时段的分析
- 截图级和时段级分析结果均支持用户编辑/自写
- UI 和日报优先展示用户自写内容，AI 分析作为 fallback

**验收条件**:
- Given 用户启动自动感知，When 到达设定间隔，Then 自动截图并保存（状态：待分析）
- Given 自动感知运行中，When 用户点击停止，Then 立即停止截图
- Given 当前时段有截图，When 距上一张截图超过 30 分钟，Then 自动结束当前时段并触发批量分析
- Given 时段分析触发，When 分析完成，Then 每张截图的分析和时段摘要均更新
- Given 用户编辑分析结果，When 保存，Then UI 和日报优先展示用户版本
- Given 用户手动触发分析，When 选择时段，Then 执行批量分析并更新结果

**优先级**: P0 (核心功能)

---

#### 6.2 闪念胶囊 (Quick Note)

**需求描述**: 全局快捷键快速记录想法，不打断工作流

**功能规格**:
- 全局快捷键 Alt+Space (可配置)
- 弹窗式输入，支持多行文本
- 可选读取当前打开的文件内容
- 记录带时间戳存入数据库

**验收条件**:
- Given 任意应用激活状态，When 按下 Alt+Space，Then 弹出速记窗口
- Given 速记窗口打开，When 输入内容并保存，Then 存入数据库并关闭窗口
- Given 用户选择读取文件，When 确认，Then 文件内容填充到输入框

**优先级**: P0 (核心功能)

---

#### 6.3 AI 日报生成 (Daily Summary)

**需求描述**: 汇总全天记录，生成结构化 Markdown 日报

**功能规格**:
- 汇总当日所有自动记录和手动记录
- AI 分析工作流模式，提取关键成果
- 生成 Markdown 格式日报
- 保存到 Obsidian Vault

**输出格式**:
```markdown
# 工作日报 - YYYY-MM-DD

## 今日概览
- 工作时长：X 小时
- 记录条数：Y 条
- 截图数量：Z 张

## 主要成果
1. ...
2. ...

## 工作时间线
- 09:00 - 10:30: ...
- 10:30 - 12:00: ...

## 待跟进事项
- ...
```

**验收条件**:
- Given 当日有记录，When 点击生成日报，Then 生成 Markdown 文件
- Given 生成成功，When 用户查看，Then 显示文件路径并提供打开选项
- Given 生成失败，When 错误发生，Then 显示具体错误信息

**优先级**: P0 (核心功能)

---

#### 6.4 截图回顾 (Screenshot Gallery)

**需求描述**: 浏览当日所有截图，点击查看大图

**功能规格**:
- 网格展示当日截图缩略图
- 点击放大查看原图
- 显示截图时间和 AI 分析内容

**验收条件**:
- Given 当日有截图，When 打开截图画廊，Then 网格展示所有缩略图
- Given 点击查看，When 点击缩略图，Then 弹窗显示大图和分析内容

**优先级**: P1 (重要功能)

---

#### 6.5 系统托盘 (System Tray)

**需求描述**: 最小化到托盘，后台静默运行

**功能规格**:
- 关闭按钮最小化到托盘
- 托盘菜单：显示窗口、退出
- 左键点击托盘图标显示窗口

**验收条件**:
- Given 应用运行中，When 点击关闭，Then 最小化到托盘而非退出
- Given 托盘图标存在，When 左键点击，Then 显示主窗口
- Given 托盘菜单，When 点击退出，Then 完全退出应用

**优先级**: P1 (重要功能)

---

#### 6.6 设置管理 (Settings)

**需求描述**: 配置应用参数

**配置项**:
| 配置项 | 类型 | 默认值 | 说明 |
|-------|------|-------|------|
| API Base URL | string | - | OpenAI 兼容 API 地址 |
| API Key | password | - | API 密钥 |
| Model | string | gpt-4o | 模型名称 |
| 截图间隔 | number | 5 | 分钟 |
| 日报时间 | time | 18:00 | 每日提醒生成日报时间 |
| Obsidian 路径 | path | - | 日报输出目录 |
| 自动捕获 | boolean | false | 启动时自动开始捕获 |

**验收条件**:
- Given 配置有效，When 保存设置，Then 存入数据库并立即生效
- Given 配置无效，When 保存，Then 显示验证错误

**优先级**: P0 (核心功能)

---

#### 6.7 工作时段管理 (Work Session Management)

**需求描述**: 将连续的截图按工作时段自动分组，以时段为单位进行上下文感知的 AI 分析

**功能规格**:
- 自动检测时段边界：两次截图间隔超过可配置阈值（默认 30 分钟）即为新时段
- 每个时段记录：开始时间、结束时间、包含的截图列表
- 时段分析：将该时段所有截图 + 上一时段上下文摘要一起发送 AI 批量分析
- AI 返回：每张截图的独立分析 + 时段整体摘要
- 用户编辑层：
  - 截图级：用户可编辑每张截图的分析内容（user_notes）
  - 时段级：用户可编辑时段摘要（user_summary）
- 优先级规则：user_notes/user_summary > AI 分析结果
- 手动触发：用户可选择任意时段手动触发（重新）分析

**验收条件**:
- Given 截图间隔超过 30 分钟，When 新截图触发，Then 创建新时段
- Given 时段结束，When 批量分析完成，Then 每张截图和时段摘要均有结果
- Given 用户编辑截图分析或时段摘要，When 保存，Then 后续展示和日报生成优先使用用户版本
- Given 用户手动选择时段触发分析，When 分析完成，Then 结果正确更新

**优先级**: P0 (核心功能)

### 7. 非功能需求

#### 7.1 性能要求

| 指标 | 目标值 |
|-----|-------|
| 应用启动时间 | <3 秒 |
| 截图处理延迟 | <2 秒 (从截图到显示) |
| AI 分析延迟 | <10 秒 (取决于网络和模型) |
| 日报生成时间 | <30 秒 (100 条记录以内) |
| 内存占用 | <200MB (空闲状态) |

#### 7.2 安全要求

- API Key 本地加密存储 (SQLite)
- 不上传用户数据到除 AI API 外的任何服务
- 截图仅本地处理和存储
- 支持本地部署 AI 模型 (通过配置 API Base URL)

#### 7.3 兼容性要求

| 平台 | 最低版本 | 截图支持 |
|-----|---------|---------|
| Windows | Windows 10 | ✅ Windows Graphics Capture API |
| macOS | macOS 11+ | ✅ xcap |
| Linux | Ubuntu 20.04+ | ✅ xcap |

#### 7.4 可用性要求

- 离线模式：无网络时本地功能正常使用
- 自动重连：网络恢复后自动重试 AI 调用
- 错误恢复：AI 调用失败时保留截图并提示重试

### 8. 技术约束

1. **Tauri v2 框架**: 必须使用 Tauri v2 的插件系统
2. **Rust 后端**: 所有核心逻辑在 Rust 端实现
3. **SQLite 数据库**: 单文件数据库，便于备份和迁移
4. **Vue 3 前端**: 使用 Composition API 和 `<script setup>`
5. **TailwindCSS**: 唯一的样式方案，无 CSS 文件

### 12. 附录

#### 12.1 数据库 Schema

```sql
-- 记录表
CREATE TABLE records (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    timestamp TEXT NOT NULL,      -- RFC3339 格式
    source_type TEXT NOT NULL,    -- 'auto' | 'manual'
    content TEXT NOT NULL,        -- JSON 或纯文本
    screenshot_path TEXT          -- 可为空
);

-- 索引优化
CREATE INDEX idx_timestamp ON records(timestamp DESC);
CREATE INDEX idx_source_type ON records(source_type);

-- 设置表 (单行)
CREATE TABLE settings (
    id INTEGER PRIMARY KEY CHECK (id = 1),
    -- AI 配置
    api_base_url TEXT,
    api_key TEXT,
    model_name TEXT,
    -- 捕获配置
    screenshot_interval INTEGER DEFAULT 5,
    change_threshold INTEGER DEFAULT 3,      -- 变化率阈值 (%)
    max_silent_minutes INTEGER DEFAULT 30,   -- 强制捕获时间
    -- 分析配置 (扩展)
    analysis_prompt TEXT,
    summary_model_name TEXT,
    summary_prompt TEXT,
    -- 输出配置
    summary_time TEXT DEFAULT '18:00',
    obsidian_path TEXT,
    auto_capture_enabled INTEGER DEFAULT 0,
    last_summary_path TEXT
);
```
